# RFC 001: Caffeine 功能对等（Parity）

## Summary

定义 Sleepless 与 [Caffeine](https://www.caffeine-app.net/) 的功能对等范围，使 Sleepless 可作为 Caffeine 的替代品，在「防止 Mac 自动休眠、熄屏与屏保」上提供相同能力与相近体验。

## Context / Problem

- Sleepless 当前仅实现：菜单栏图标、点击切换防休眠、右键菜单（状态标题、切换、退出）。
- 用户若从 Caffeine 迁移，会期望：开机自启、启动时是否默认激活、可选的定时关闭、以及从菜单快速设置临时时长。
- 需要一份明确的功能对等文档，作为实现与验收依据。

## 参考：Caffeine 行为与选项（学习自官方与社区）

| 能力 | Caffeine 行为 |
|------|----------------|
| **菜单栏** | 咖啡杯图标；点击切换开/关；满杯=防休眠，空杯=允许休眠。 |
| **右键/⌘+点击** | 打开菜单：状态、**偏好设置**、**临时时长（Timeout）**、退出。 |
| **偏好设置** | 1）**开机启动**：每次登录时启动应用；2）**启动时激活**：启动后是否默认开启防休眠；3）**默认时长**：防休眠在多久后自动关闭（如 5 分钟、1 小时），或「不自动关闭」。 |
| **临时时长（菜单）** | 从菜单选择「仅防休眠 X 分钟/小时」或「 indefinitely」，无需进偏好。 |
| **系统行为** | 防休眠时：阻止系统空闲休眠、屏幕变暗、屏保启动（与 IOKit `NoIdleSleep` + `NoDisplaySleep` 一致）。 |
| **外观** | 仅菜单栏，无 Dock 图标；偏好通过菜单打开。 |

## Goals（对等目标）

1. **菜单栏交互**
   - 单击图标：切换「防休眠」开/关。
   - 图标状态：防休眠中 vs 允许休眠，视觉区分明确（与当前月牙图标一致即可）。
   - 右键或 ⌘+点击：显示菜单。

2. **右键/⌘ 菜单内容**
   - 当前状态行（只读，如「正在防止休眠」/「允许休眠」）。
   - **偏好设置…**：打开偏好窗口。
   - **临时时长** 子菜单：如「5 分钟」「15 分钟」「30 分钟」「1 小时」「2 小时」「不自动关闭」；选择后开启防休眠并在对应时间后自动关闭（若当前未开启则先开启）。
   - 切换防休眠（可选，与单击重复，可保留以便键盘操作）。
   - 退出。

3. **偏好设置（窗口）**
   - **开机时启动 Sleepless**：勾选后加入登录项（Launch at login）。
   - **启动时自动开启防休眠**：勾选后 app 启动即调用防休眠，否则仅显示图标、不阻止休眠。
   - **默认防休眠时长**：下拉或单选——例如「5 分钟」「15 分钟」「30 分钟」「1 小时」「2 小时」「不自动关闭」；仅影响「通过单击或菜单“切换”开启」时的默认定时行为；若选「不自动关闭」，则与当前行为一致（需用户手动关或选菜单中的临时时长）。

4. **防休眠与定时**
   - 防休眠实现保持现有 IOKit 方式（NoIdleSleep + NoDisplaySleep）。
   - 定时关闭：选择「X 分钟/小时」后，在对应时间后自动调用「允许休眠」并更新图标/菜单状态；若用户在此前手动关闭防休眠，则取消该定时器。
   - 「临时时长」与「默认时长」共用同一组时长选项，逻辑一致。

5. **非目标（明确不做对等）**
   - 不要求与 Caffeine 完全一致的 UI 文案或图标风格。
   - 不承诺与 Caffeine 相同的快捷键（可后续单独 RFC）。
   - 多语言/本地化不在本 RFC 范围，可后续补充。

## Proposed Solution（实现要点）

- **状态与定时器**：在应用内维护「是否防休眠」与「当前定时器」（可选）。菜单与图标根据状态 + 定时器剩余时间更新。
- **偏好持久化**：使用 `UserDefaults` 或 `AppStorage` 存：是否开机启动、是否启动时激活、默认时长（枚举或分钟数）。
- **登录项**：使用 `SMAppService`（macOS 13+）或 `LSSharedFileList` 等 API 实现「开机时启动」。
- **偏好 UI**：SwiftUI 设置窗口（已预留 `Settings { … }`），包含上述三个选项；可从菜单「偏好设置…」用 `NSApp.sendAction(#selector(NSApplication.showPreferencesWindow(_:)), to: nil, from: nil)` 或等价方式打开（若需兼容性则用 `openSettings` / 展示 Settings 场景）。
- **临时时长子菜单**：在 `NSMenu` 下添加子菜单，每项对应一个时长或「不自动关闭」；选择时：若未防休眠则先开启防休眠，再设置定时器；若已在防休眠则仅更新/重置定时器。

## Implementation Details（可后续拆任务）

| 项目 | 说明 |
|------|------|
| 数据模型 | 偏好：`launchAtLogin: Bool`, `activateOnLaunch: Bool`, `defaultDuration: DefaultDuration`（枚举：5m, 15m, 30m, 1h, 2h, indefinite）。 |
| 定时器 | 使用 `Timer` 或 `DispatchWorkItem`，在选定分钟后取消防休眠并更新 UI。 |
| 启动流程 | `applicationDidFinishLaunching` 中：若 `activateOnLaunch` 则调用 `preventSleep()`；若 `defaultDuration != .indefinite` 则启动对应定时器。 |
| 登录项 | `SMAppService.mainApp`（macOS 13+）或回退到 `LSSharedFileListInsertItemNamed` 等。 |
| 设置窗口 | SwiftUI 表单 + `@AppStorage` / `UserDefaults`，并确保从菜单可打开。 |

## Alternatives Considered

- **不提供定时**：仅做「开/关」，实现简单但无法达到 Caffeine 的「临时防休眠」体验，故不采纳。
- **仅做“默认时长”不做“临时时长”**：会缺少从菜单快速设时长的能力，与 Caffeine 不一致，故不采纳。

## Risks and Concerns

- 定时器在 app 被系统暂停时的行为（如合盖）：需明确「防休眠」在合盖后是否仍由系统维持；定时器可在 app 回到前台时做一次校正（剩余时间重算或直接按“已到期”处理）。
- 登录项权限：需在首次勾选「开机启动」时处理权限或提示（若系统有）。

## Testing Strategy

- 手动测试：开机启动、启动时激活、默认时长到期自动关闭、菜单临时时长开启并到期关闭、手动关闭后定时器取消。
- 回归：现有「单击切换、右键菜单、退出时释放防休眠」仍正常。
